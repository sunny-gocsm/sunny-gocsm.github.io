The problem & the wedge

①Onboarding is where agencies silently bleed revenue

63%
of buyers weigh onboarding in the purchase decision — and poor onboarding is the #1 churn driver.
Spreadsheets & Slack
No shared source of truth for "fully onboarded", no visibility into where a client is stuck.
Margin loss
An unactivated sub-account is pure loss — the agency pays HighLevel and collects nothing.
⚡

Why GoCSM, why now. DAPs (Userpilot/Pendo) do in-app guidance but no accountability; onboarding platforms (Rocketlane/GUIDEcx) do accountability but live outside the product. GoCSM does both — because it already runs inside HighLevel via Custom-JS injection, which is exactly what makes automatic step-completion detection ("I did it and GoCSM just knew") feasible. A category we can own.

Solution architecture

Each layer is a distinct surface and user, but all three read/write the same per-step completion record.

The contract

④One step, nine states — rendered identically everywhere

All three surfaces render the same state grammar. stalled is a computed agency-side lens, never stored, never shown to the client.

not_started 🔒 locked in_progress verifying waiting_on_agency needs_attention ✓ done skipped stalled · lens

Versioned by design

Editing a published journey creates a new draft version; clients are pinned to the version they started on. Publishing affects new assignments only; migrating in-flight clients is an explicit action with stated blast radius. (Agencies never see version numbers — just Draft / Live / Editing.)

Pluggable detector per step

detector.type ∈ {auto_signal, manual_confirm, inbound_webhook}. Never a generic boolean. Every detector ships a confidence model + an obvious manual override — auto-detection that's wrong is worse than manual. last_intervention closes the loop ("Nudged 2d ago — no movement").

The detector framework — the differentiator

⑤Three detectability tiers

A Cleanly auto-detectable

A clear state change we can read. The demo — "I did it and GoCSM just knew." Auto-detect; manual override as safety net.

B Detectable but ambiguous

A laggy/flaky proxy (DNS, carrier review). Auto-detect the proxy; manual override both directions + a confidence indicator.

C Manual-only

No reliable signal (book a call, watch a video). Manual check; optional soft signal (watch-%, booking webhook).

🔀

Non-blocking model. verifying (awaiting carrier/DNS) never blocks an independent step — the client is advanced through the ~11 of 15 independent steps in parallel. locked propagates only from genuine hard dependencies: send-SMS ← A2P campaign ← A2P brand ← phone. "Current step" = next actionable step, never a frozen wait screen.

Layer A — the agency builder · redesigned 30 Jun

⑦A guided, template-first wizard — with a live preview

One decision per screen, smart defaults pre-selected, the client experience building live beside you, and full power as progressive depth behind "Customize". Value in minutes; the blank canvas is never the first thing they see. (Replaced the old 9-step builder + gated editor — see Changes made on 30 June.)

1Template› 2Steps› 3Order› 4Experience› 5Look & feel› 6Review + live client preview, pinned right

🧩 Template-first, power on demand

Start from a real template (Standard 15 / Fast-track 7 / Local services 9) or blank. Each step's inline Customize restores title, client copy, completion method, video and per-asset sub-steps — capability hidden, not removed.

✓ Auto-verify is locked (the wedge)

Catalog steps are always GoCSM-auto-verified — no picker (agencies can't downgrade our tracking). The 4-way method picker (auto / client-confirms / web-event / for-reference) shows only for custom steps. Every row carries an "Auto-verified" badge.

🧮 Asset steps: any vs specific

Workflows · funnels · forms · pipelines default to "any"; "specific" picks a remembered snapshot sub-account + one named asset via a searchable dropdown. Re-addable, so 3 workflows = 3 steps.

⏱ One time number

The agency sets one thing: "onboarded within __ days" (default 14) — the promise they already make. Per-step thresholds are silent system defaults; "SLA" never appears.

📍 Look & feel

Visual placement mockups (Floating · Top banner, default · Menu · Embedded "Launchpad") + brand colour auto-detected ("✦ Pulled from your brand"). Banner is default to avoid colliding with the agency's existing support widget.

🔁 Returning agency → summary

A calm "Live" overview (JourneySummary); each row "Edit ›" jumps into the wizard at the right step; a quiet escape opens the advanced JourneyEditor for deep-links, plan variants & granular config.

🎨

White-label is the moat. Agency logo + accent (pre-filled from HighLevel), a maintained white-label tutorial video set (per-step override), and Tracking-only mode (detection/queue/automations run, no client widget). GoCSM brand never appears on the client surface.

Layer B — the client doer

⑧A competent guide — a map, not a locked ladder

One action, friendly progress

A compact card overlaying HL (not a full pane): logo + "Your setup checklist" + thin bar + "3 of 8 done". Progressive disclosure — Done group on top, the suggested step as hero, "More steps" + "Waiting on review" collapsed. Encouragement, never a grade (no %, no journey-day, no stall language).

Act on any step, any order

The hero is a suggestion, not a gate. Completion is position-independent — a step done out of order (even directly in HL) flips to ✓. The only thing that locks is the carrier chain (A2P campaign ← brand ← phone); everything else is always available.

✨

The signature mechanic — "it just knew". On return from a deep link, the widget re-verifies in under a second. Tier A snaps to ✓ (no spinner); Tier B enters verifying with honest "carrier review usually 1–3 days" copy + a forward path ("Meanwhile, let's set up {next} →"). A rejection flips only that step to needs_attention with one fix — all other progress is preserved (recoverable partial failure).

⑨Not a report — a work queue with evidence

Answers one question every morning: who do I save today, and did yesterday's saves work? Top to bottom, "needs me" is loud and first; "in flight" is calm and below.

Two clocks, operator-only. Journey clock (Day N vs target) = "how far along?"; step clock ("Stuck 12 days") = "stuck right now?". Neither appears on the client doer. ⑪Phasing — each phase independently releasable

P1
Builder + detector framework

Lifecycle (draft/publish/version/pin), templates, guided wizard, fine-tune editor, theming. Detector contract + 2–3 Tier-A detectors. Schema & versioning are the long poles.

P2
Doer + contextual video

Injected widget, white-label + client voice, deep-links, re-verify-on-focus, nine-state rendering, graduation. First end-client value.

P3 · Overview shipped 30 Jun
Overview + queue + trigger

Dollar-led verdict, triage-strip-first tracker, money-ranked queue w/ undo, relief strip, Hub lens, onboarding.step.stalled. Redesign live in prototype.

P4
Sherlock assist

Conversational, GHL-specific stuck-diagnosis with concrete pointers. Built last, after observing real stall patterns.

🪤

Sequencing trap. Don't ship P1–P2 "manual-only, add detection later" — retrofitting detection onto a boolean schema is painful and a manual-only product loses to Trello. The schema must assume detectors and version-pinning from day one even if implementation lags.
